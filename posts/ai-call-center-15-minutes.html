Building Your Own AI Call Center in 15 Minutes with Asterisk and Python

Published on November 6, 2025 | 8 min read

Sounds like a months-long project, right? Wrong. With the Asterisk AI Voice Agent, you can deploy a production-ready system in under 15 minutes.

In this tutorial, I'll walk you through the entire process, from zero to a working AI voice agent that can answer calls and hold intelligent conversations.

What You'll Build

By the end of this tutorial, you'll have:

  • A fully functional Asterisk PBX integrated with AI services
  • Real-time speech-to-text (STT) and text-to-speech (TTS) capabilities
  • An intelligent conversational agent powered by modern LLMs
  • A Docker-based deployment that's easy to manage and scale

Prerequisites

Before we begin, make sure you have:

  • Docker and Docker Compose installed on your system
  • An Asterisk server (version 16 or later) with ARI enabled
  • API keys for your chosen AI services (OpenAI, Deepgram, or ElevenLabs)
  • Basic command-line knowledge
Pro Tip: If you don't have an Asterisk server yet, you can use the included Docker Compose setup that bundles everything together.

Step 1: Clone the Repository

First, grab the code from GitHub:

git clone https://github.com/hkjarral/Asterisk-AI-Voice-Agent.git
cd Asterisk-AI-Voice-Agent

Step 2: Configure Your Environment

The project uses environment variables for configuration. Copy the example file and edit it with your API keys:

cp .env.example .env
nano .env

Here's what a minimal configuration looks like:

# Asterisk Configuration
ASTERISK_HOST=localhost
ASTERISK_ARI_PORT=8088
ASTERISK_ARI_USER=asterisk
ASTERISK_ARI_PASSWORD=asterisk

# AI Service Configuration
STT_PROVIDER=deepgram
DEEPGRAM_API_KEY=your_deepgram_key_here

LLM_PROVIDER=openai
OPENAI_API_KEY=your_openai_key_here

TTS_PROVIDER=elevenlabs
ELEVENLABS_API_KEY=your_elevenlabs_key_here
Privacy Note: You can also use local models (Vosk for STT, Llama for LLM, Piper for TTS) if you prefer to keep everything on-premise. Just change the provider settings accordingly.

Step 3: Launch with Docker Compose

This is where the magic happens. With a single command, Docker Compose will:

  • Pull the necessary images
  • Set up the Python application
  • Connect to your Asterisk server
  • Start the AI voice agent
docker-compose up -d

Watch the logs to ensure everything starts correctly:

docker-compose logs -f

You should see output indicating that the agent has successfully connected to Asterisk's ARI interface.

Step 4: Configure Asterisk Dialplan

Now we need to tell Asterisk to route calls to our AI agent. Add this to your extensions.conf:

[from-internal]
exten => 100,1,NoOp(AI Voice Agent)
 same => n,Stasis(ai-voice-agent)
 same => n,Hangup()

Reload the Asterisk dialplan:

asterisk -rx "dialplan reload"

Step 5: Make Your First Call

That's it! You're ready to test. From any phone connected to your Asterisk server, dial extension 100. You should hear the AI agent greet you and start a conversation.

Try asking it questions like:

  • "What services do you offer?"
  • "Can you tell me your business hours?"
  • "I'd like to schedule an appointment."

The agent will respond in real-time, using the LLM to generate contextually appropriate answers.

Customizing Your Agent

The default agent is designed to be a helpful assistant, but you can easily customize its behavior by editing the system prompt in src/config.py:

SYSTEM_PROMPT = """
You are a friendly customer service agent for Acme Corp.
Your role is to answer questions about our products and services.
Always be polite, concise, and helpful.
If you don't know something, admit it and offer to transfer to a human.
"""

Restart the container to apply changes:

docker-compose restart

Monitoring and Debugging

The agent logs all conversations and system events. To view them:

docker-compose logs -f ai-voice-agent

For more detailed debugging, you can enable verbose logging by setting LOG_LEVEL=DEBUG in your .env file.

What's Happening Under the Hood?

When a call comes in, here's the flow:

  1. Asterisk receives the call and triggers the Stasis application
  2. The Python agent connects via ARI and creates an ExternalMedia channel
  3. Audio streams in real-time between Asterisk and the agent
  4. Speech-to-text converts the caller's voice to text
  5. The LLM generates an intelligent response
  6. Text-to-speech converts the response back to audio
  7. The audio is played back to the caller through Asterisk

All of this happens in near real-time, with typical latencies under 500ms for cloud services and under 2 seconds for local models.

Troubleshooting Common Issues

Agent won't connect to Asterisk

Check that ARI is enabled in ari.conf and that the credentials in your .env file match.

No audio during calls

Ensure that the ExternalMedia application is properly configured and that your firewall allows RTP traffic.

High latency

If you're using cloud services, check your internet connection. For local models, ensure you have adequate CPU/GPU resources.
