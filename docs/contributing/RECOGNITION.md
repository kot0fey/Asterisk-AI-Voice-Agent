## Tiers

### Contributor

**Requirement:** 1+ merged PR (any type)

**Recognition:**
- Name and avatar in [CONTRIBUTORS.md](../../CONTRIBUTORS.md)
- Acknowledged in release notes for the version containing your contribution

### Operator Contributor

**Requirement:** Shared deployment docs, working configs, case studies, or dialplan examples

**Recognition:**
- Everything in Contributor tier
- Special "Operator Contributor" badge in CONTRIBUTORS.md
- Your deployment story featured in our case studies

### Core Contributor

**Requirement:** 5+ merged PRs across multiple areas

**Recognition:**
- Everything in previous tiers
- Listed with area of expertise in CONTRIBUTORS.md
- Discord role: "Core Contributor"
- Invited to monthly project calls
- Input on roadmap priorities

## Contribution Types We Recognize

| Type | Emoji | Description |
|------|-------|-------------|
| Code | `code` | Bug fixes, features, adapters |
| Documentation | `doc` | Guides, tutorials, corrections |
| Tests | `test` | Unit tests, integration tests |
| Ideas | `ideas` | Feature proposals, architecture suggestions |
| Bug Reports | `bug` | Detailed issue reports with repro steps |
| Configs | `example` | Working configurations shared with community |
| Translations | `translation` | Setup guides in other languages |
| Mentoring | `mentoring` | Helping others contribute |
| Review | `review` | PR reviews and feedback |

## How Recognition Works

When your PR is merged, a maintainer will add you to CONTRIBUTORS.md. If you're not added within a week, ping us on [Discord](https://discord.gg/ysg8fphxUe) or open an issue.

Non-code contributions (case studies, configs, dialplan docs) get equal visibility. You don't need to write Python to be a valued contributor.
